# 顔の好み診断 (Face Preference Analyzer)

自分専用の「顔の好み」を、**2 枚の顔を見比べてどちらが好きかを選ぶ**だけで
学習・可視化するツール。絶対評価（点数付け）ではなくペア比較を採用している
ため、基準がブレにくく安定したランキングが得られる。

## 仕組み（MVP）

```
顔画像(data/faces/) ──→ Gradio で「どっちが好き?」出題
                              │ クリックで (勝者, 敗者) を記録
                              ▼
                        SQLite 比較ログ
                              │
                              ▼
              Bradley-Terry モデルで各顔の潜在好み度を推定
                              │
                              ▼
                    あなたの好みランキング表示
```

## セットアップ

```bash
py -3 -m pip install -r requirements.txt
```

## 使い方

1. 評価用の顔画像を用意する
   - 合成顔を自動取得（推奨・ライセンス手続き不要）:
     ```bash
     py -3 download_faces.py --count 200
     ```
   - もしくは FFHQ / CelebA 等の画像を `data/faces/` に直接配置
2. アプリを起動:
   ```bash
   py -3 app.py
   ```
3. ブラウザ（http://127.0.0.1:7860）で好きな方を選び続ける
4. 数十回選んだら「ランキングを更新」で好みトップを表示

## ファイル構成

| ファイル | 役割 |
|---|---|
| `app.py` | Gradio のペア比較 UI と比較ログ記録 |
| `ranking.py` | Bradley-Terry による好み度推定（外部依存なし） |
| `download_faces.py` | 評価用の合成顔をダウンロード |
| `data/faces/` | 評価対象の顔画像（git 管理外） |
| `data/comparisons.db` | 比較ログ（git 管理外） |

## ロードマップ

- [x] フェーズ1: ペア比較 UI + Bradley-Terry ランキング（本MVP）
- [ ] フェーズ2: 顔の特徴量（embedding / 幾何属性）で**未評価の顔**の好み度を予測
- [ ] フェーズ3: 属性ベースの説明（SHAP で「なぜ好きか」を言語化）
- [ ] フェーズ4: 能動学習で少ない回答数でも高精度化

## メモ

合成顔（thispersondoesnotexist）は実在人物ではないため肖像権・プライバシー
の懸念が小さい。実在の公開データセットを使う場合は各データセットのライセンス
・利用規約を確認すること。
